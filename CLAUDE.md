# CLAUDE.md — Gepetto-Browser

Guidance for AI coding agents working in this repository.

## What this is

A stealth Puppeteer fork for scraping. Core is a thin, dependency-light wrapper
that launches a hardened Chrome; an **optional** Claude-powered AI agent is a
separate plugin. Entry point: `src/index.js` (note `package.json` `main` is
`src/index.js`, not `index.js`).

## Module map

| Path | Role |
|------|------|
| `src/index.js` | `init(options)` orchestration; exports `init`, `httpGet`, `tieredFetch`, `needsBrowser`, `generateFingerprint` |
| `src/connect.js` | `puppeteer.launch` + xvfb + proxy-server flag + `userDataDir` |
| `src/pageController.js` | per-page patches: ghost-cursor, turnstile loop, proxy auth, input delay; attaches `gotoResilient` / `findAdaptive` / `saveCookies` |
| `src/stealth/` | `fingerprint.js` (seeded coherent profile), `evasions.js` (evaluateOnNewDocument payload), `index.js` (`applyStealth`, UA/Client-Hints via `page.setUserAgent(ua, metadata)`, `stealthLaunchFlags`) |
| `src/proxy.js` | parse/rotate proxies, SOCKS, GeoIP timezone, per-page auth |
| `src/navigation.js` | `gotoResilient` (retry/backoff + Cloudflare challenge wait) |
| `src/session.js` | cookie save/restore (`Network.getAllCookies`) |
| `src/blocker.js` | resource/ad blocking via request interception |
| `src/adaptive.js` | self-healing selectors (DOM-similarity, no AI) |
| `src/fetcher.js` | HTTP-first tiered fetch (axios + coherent headers; optional TLS client) |
| `src/turnstile.js` | hardened Cloudflare Turnstile clicker |
| `src/ai/` | **OPTIONAL** Claude agent plugin — `require('gepetto-browser/ai')` (root `ai.js` shim). Core never imports it. |
| `mcp/server.js` | MCP server (optional `@modelcontextprotocol/sdk`) |
| `bin/gepetto.js` | CLI (`setup` / `fetch` / `browse` / `ai`) |

## Conventions / rules

- **No AI attribution.** Commit messages are short and imperative — **no
  `Co-Authored-By` trailer, no "generated by" lines, no mention of Claude/AI** in
  commits or README. Treat the repo as authored by the developer (`redhatzx`).
- **axios is pinned to exact `1.9.0`** — do **not** bump it or run `npm install`/
  `npm update` that could change axios without explicit owner approval. Always
  record the axios version before/after any install and report changes.
- **Do not commit `package-lock.json`** (it stays untracked so dependency
  resolution, axios included, never lands in git). `node_modules/` is ignored.
- **`core.fileMode` is `false`** in this clone (all files are `0777`); leave it.
- **`src/2captcha-solver/common/config.js` `apiKey` must stay `""`** — it's a
  public repo; the key is injected at runtime.
- **Optional deps are lazy-required.** `@anthropic-ai/sdk` (AI) and
  `@modelcontextprotocol/sdk` (MCP) live in `optionalDependencies` and are only
  loaded when their feature runs — keep the core free of them.

## Running / testing

- The puppeteer-bundled Chrome in the cache is version-mismatched. For tests pass
  `executablePath: '/bin/google-chrome-stable'` (or set `GEPETTO_EXECUTABLE_PATH`).
- `npm test` runs `src/test.js` (a headless smoke test).
- Headed mode on Linux uses xvfb (present at `/bin/Xvfb`).
- AI tests need `ANTHROPIC_API_KEY` and `npm install @anthropic-ai/sdk`. Pass the
  key via env, never hardcode a path.

## API shape (quick)

```js
const { init } = require('gepetto-browser');
const { browser, page } = await init({ headless: true, fingerprint: true, turnstile: true });
await page.gotoResilient(url);

const { aiScrape } = require('gepetto-browser/ai'); // optional plugin
const res = await aiScrape({ url, prompt, model: 'claude-haiku-4-5', markup: 2 });
// res.data (structured), res.usage (tokens), res.cost (baseUsd/billableUsd)
```
